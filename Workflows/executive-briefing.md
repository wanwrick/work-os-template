# Executive Briefing Workflow

> Preparation guide for board decks, C-suite presentations, and leadership updates. Based on Pyramid Principle, SCQA, and executive communication frameworks.

---

| Question | Answer |

### Audience Adaptation
| If Audience Is... | Lead With... | Depth | Format |
|-------------------|-------------|-------|--------|
| CTO | Architecture, tech risk, innovation | Deep on strategy | Diagrams + metrics |
| VP Engineering | Velocity, reliability, team health | Moderate | Dashboards + status |
| VP Product | Business value, user impact, timeline | Outcome-focused | SCQA + roadmap |
| CFO/Finance | ROI, cost, budget adherence | Numbers-first | Tables + trends |
| Board | Strategic direction, risk, competitive | High-level only | Narrative + 3 key slides |

---

## Step 2: Structure the Narrative

### Pyramid Principle (Top-Down)
```
          [Key Message / Recommendation]
         /              |              \
   [Reason 1]     [Reason 2]     [Reason 3]
   /    \          /    \          /    \
[Data] [Data]  [Data] [Data]  [Data] [Data]
```

**Rule:** Start with the answer. Support with 3 reasons max. Each reason backed by evidence.

### SCQA Flow
1. **Situation:** [Shared context: what the audience knows] (1 slide)
2. **Complication:** [What changed: the tension] (1 slide)
3. **Question:** [What should we do?] (implied or explicit)
4. **Answer:** [Your recommendation with evidence] (2-3 slides)
5. **Next Steps:** [Clear actions, owners, timeline] (1 slide)

---

## Step 3: Build the Deck

### Slide Budget (10 slides max)
| Slide | Purpose | Time |
|-------|---------|------|
| 1 | Title + one-line key message | 30 sec |
| 2 | Situation: current state | 1 min |
| 3 | Complication: what changed | 1 min |
| 4 | Recommendation: your answer | 2 min |
| 5-6 | Evidence: data, analysis | 3 min |
| 7 | Risk and mitigation | 1.5 min |
| 8 | Implementation plan | 1.5 min |
| 9 | Ask: decision or resources needed | 1 min |
| 10 | Appendix (backup detail) | Only if asked |

### Slide Design Rules
- Title = conclusion (not topic). "Revenue grew 15%" not "Revenue Update"
- One message per slide
- 70% visual, 30% text
- No bullet points: use diagrams, charts, tables
- Consistent template and fonts

### Anticipate the Top 3 Questions
| Question | Your Answer | Supporting Data |
|----------|-------------|-----------------|
| 1. [Hardest question] | | |
| 2. [Budget/resource question] | | |
| 3. [Risk/timeline question] | | |

### Q&A Protocol
1. **Pause**: 2 seconds before answering
2. **Rephrase**: confirm you understood
3. **Answer concisely**: 30 seconds max
4. **Bridge**: connect back to your key message
5. If you don't know: "I'll get you that by [specific date]"

---

## Step 5: Delivery Checklist

### Before the Meeting
- [ ] Dry run with a peer (get feedback on flow and timing)
- [ ] Pre-read sent to attendees 24 hours in advance
- [ ] Backup slides ready in appendix
- [ ] Tech check (screen sharing, projector)
- [ ] Know your opening line cold

### During the Meeting
- [ ] Start with the key message (BLUF)
- [ ] Watch the room: adapt pace to energy
- [ ] Use Power of Pause after key points (2 seconds)
- [ ] Language upgrades: "I recommend" not "I think"
- [ ] End 5 minutes early for discussion
- [ ] Close with clear next steps and owners

### After the Meeting
- [ ] Send written summary within 2 hours
- [ ] Include: decisions made, action items, owners, deadlines
- [ ] Follow up on any "I'll get back to you" commitments
- [ ] Log decisions in `Knowledge/Decisions/`
